* 머신러닝이란?
- 머신 러닝은 분석 모델 구축을 자동화하는 데이터 분석 방법이다.
- 데이터를 반복적으로 학습하는 알고리즘을 사용하는 머신러닝을 통해 컴퓨터는 어디를 볼 것인지 명시적으로 프로그래밍하지 않고도 숨겨진 인사이트를 찾을 수 있다.
- 알고리즘의 수행 작업을 명시적으로 지정할 필요가 없다는 것이 핵심 아이디어이다.
(알고리즘은 데이터를 반복적으로 학습한다.)
* 머신러닝의 활용
- 부정 적발, 장비 오류 예측, 이메일 스팸 필터링, 재무 모델링, 텍스트 감성 분석, 추천 엔진 등 수 많은 응용 프로그램에 사용된다.

* 머신러닝의 프로세스
[순서 1] Data Acquisition(DAQ) : 데이터를 측정하여 수집 및 처리하는 모든 과정, 즉 데이터 습득
[순서 2] Data Cleaning : 누락된 값이 있는 경우 이를 채우거나 삭제
[순서 3] 데이터를 학습 세트와 테스트세트로 나눈다.
이유 : 학습 데이터를 이용해 모델을 학습하기 때문에
[3-1] Test Data
[3-2] Model Training & Building
이후에 model testing에서 다시 반복해야 한다면 3-2로 이동한다.
[순서 4] 머신 러닝 모델에 가장 적합한 매개변수를 찾을 때까지 순서 3을 반복한다.
[순서 5] 모든 준비가 끝나면 해당 모델을 새로운 데이터에 배포

* 다양한 유형의 머신러닝
- Spark MLlib 라이브러리 : 지도 및 비지도 학습 테스트용으로 설계
- 대부분 알고리즘은 위의 지도 및 비지도 학습 두 범주에 포함됨
(1) 지도 학습 : 원하는 결과값이 있을 경우 정해진 입력값(즉, 라벨)이 지정된 예시를 사용해 학습
 (ex1: 장비에는 failed에 대해 "F", 또는 runs에 대해 "R"이라는 라벨이 지정된 데이터 포인트 존재)
 (ex2: 주택 가격 데이터- 방 개수, 위치, 평방 피트 등과 같은 특징 뿐만 아닌, 주택 가격이라는 라벨 존재)
 > 지도 학습 알고리즘은 각 라벨의 올바른 결과값과 일련의 입력값을 수신한다.
 (알고리즘은 실제 출력값을 올바른 출력과 비교하여 오류를 찾는 방식으로 학습한 다음 그에 따라 모델을 수정한다.)
 > 분류, 회귀, 예측 및 gradient boosting과 같은 방법을 사용해 패턴을 파악하여 라벨이 지정되지 않은 추가 데이터의 라벨 값을 예측할 수 있다.
 > 지도학습은 주로 라벨이 지정된 과거 데이터를 이용해 미래 이벤트를 예측하는 응용프로그램에서 사용됨
(2) 비지도 학습: 모든 과거 데이터가 있지만 이에 대한 라벨이 없는 경우 비지도 학습을 사용한다.
 ex) 시스템 : 올바른 값을 알려주지 않는다.
 > 알고리즘을 스스로 파악해야 함. 이때 목표는 데이터를 탐색하고 데이터 내에서 일종의 구조를 찾는 것
 > ex) 고객의 데이터를 다루는 경우 고객 세그먼트를 구분하는 주요 속성을 찾을 수 있다.
 > 주요 기술 : 자기 조직화 지도, 최근접 이웃 지도, K-평균 클러스터링, 특이값 분해
 > 문제점 : 비지도 학습 모델의 결과 평가가 쉽지 않다. (이유: 애초에 데이터 라벨이 존재하지 않기 때문!!)
 > 이러한 라벨을 찾거나 데이터 내에서 일종의 구조를 찾기 위해 어떤 방식으로든 라벨을 클러스터링하려고 하기 때문에 그 결과를 비교할 수가 없다.
